ESP32 打印验证

手机开热点 → ESP32 连上 → 用本页面把文本发给 ESP32 → 看串口和打印机出纸。 本页面用顶层跳转触发 GET，能从 https:// 站点直接调 ESP32 的 HTTP 接口。 发送前自动 UTF-8 → GBK 转码（codepage 936），ESP32 端 FS & 进入中文模式后透传给打印机。

等待操作。
为什么用"顶层跳转"而不是 fetch？

浏览器禁止 HTTPS 页面用 fetch() 调 HTTP 资源（混合内容规则）， 但允许 HTTPS 页面"导航"到 HTTP 地址（只会显示"不安全"提示）。 所以本页面构造 http://<ip>/print?text=... 然后 window.location 跳过去， 让 ESP32 解析 query 参数后打印，并返回一个"可打印"确认页。

本地开发（http://localhost:5173/print-test.html）下点 Ping 按钮，能用 fetch 验通； 线上 HTTPS 下 Ping 会被拦截属正常，请直接点"发送"。
